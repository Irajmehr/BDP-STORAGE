WHAT THIS STEP COVERS:
The Lot Attributes page defines every available attribute in the system. Each attribute has a Code, Description, Type (Option, Text, Date, Integer, or Decimal), predefined Values for Option types, and a "Mandatory on Inbound Entry" flag that enforces data capture at receiving.

DETAILED CONTEXT:
Making COO mandatory on inbound means no one can post a purchase receipt without entering the country of origin — critical for seafood import compliance. The Type field controls validation: Option restricts to predefined values, Date enforces date format, and Integer/Decimal enforce numeric entry.

KEY NARRATION POINTS:
The Lot Attributes page — sometimes called the master list — is where you define every attribute available in the system. Each attribute has a Code, a Description, and a Type that controls what kind of data it accepts. Option type restricts entry to predefined values you configure — for MYGRADE, that might be MyG_A, MyG_B, and MyG_C. Text type allows free-form entry, useful for attributes like NOTES or SUPPLIER_REF where values are unpredictable. Date type enforces date format, perfect for EXPIRATION or HARVEST_DATE.
