TOPIC: "Lot Attribute Setup" — Step 6: Matrix Configuration Columns

WHAT THIS STEP COVERS:
Each row in the Item Lot Attributes matrix has configuration columns: Pre-Pricing (auto-adjust cost by attribute), Dimension Code (post to specific GL dimensions), Transformation Rule (convert attribute values), Fixed per Rec. No. (lock value per receipt), and Mandatory on Inbound (require entry at receiving).

DETAILED CONTEXT:
Pre-Pricing enabled on MYGRADE means the system automatically adjusts the purchase price based on the grade received — Grade A gets the premium price, Grade B gets the discount price, all without manual intervention. Dimension Code links attributes to financial reporting dimensions.

KEY NARRATION POINTS:
Each row in the Item Lot Attributes matrix carries configuration columns that control how that specific attribute combination behaves in transactions. Pre-Pricing is one of the most impactful: when enabled on the MYGRADE attribute, the system automatically adjusts the purchase price based on the grade received. If the vendor ships Grade A, the PO line gets the Grade A premium price. If the inspection downgrades it to Grade B, the price adjusts automatically — no manual correction needed. Dimension Code links an attribute combination to a financial dimension, so Grade A Norwegian salmon can post to a different cost center than Grade B Chilean salmon, giving your finance team attribute-level profitability reporting without extra journal entries.
