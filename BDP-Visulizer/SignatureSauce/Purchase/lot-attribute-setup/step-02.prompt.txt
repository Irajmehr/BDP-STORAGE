WHAT THIS STEP COVERS:
Option Attributes 1 through 10 are used for lot-level option value tracking such as BRAND, PACK TYPE, or CERTIFICATION. Unlike Global Attributes which define the matrix structure, Option Attributes capture additional lot metadata used for filtering, reporting, and customer allocations.

DETAILED CONTEXT:
BRAND as Option Attribute 1 lets you track whether a lot of salmon is "Happy Fish" or "Norlax" brand — useful for customer-specific allocations and reporting. Option Attributes do not create matrix rows; they attach to lots as supplementary data.

KEY NARRATION POINTS:
Option Attributes serve a different purpose than Global Attributes. Where globals define the matrix structure, Option Attributes capture supplementary lot-level data used for filtering, reporting, and customer-specific allocations. The setup page provides ten option attribute slots. A common use is BRAND — tracking whether a lot of frozen salmon fillets is the "Happy Fish" brand or the "Norlax" brand. Both might be Grade A from Norway, so they share the same global attribute combination, but the brand distinction matters when a customer contract specifies one brand over another.
